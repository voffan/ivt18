Правила оформления исходного кода:

1. Именование Классов:
   Использовать латиницу.
   Название не должно содержать названия стандартных функций
   Наименование классов должно нести смысловую нагрузку и должно отображать сущность, которую класс реализует.
   При написании наименования следует придерживать UpperCamelCase

2. Именование переменных/данных(поле) класса
   Использовать латиницу.
   Наименование переменных должно нести смысловую нагрузку и должно отображать сущность, которую он реализует.
   При написании наименования следует придерживать lowerCamelCase

3. Именование свойств класса:
   Использовать латиницу.
   Название не должно содержать названия стандартных функций
   Наименование свойств классов должно нести смысловую нагрузку и должно отображать сущность, которую он реализует.
   При написании наименования следует придерживать UpperCamelCase

4. Именование функций/методов класса:
   Использовать латиницу.
   Название не должно содержать названия стандартных функций
   Наименование функций/методов должно нести смысловую нагрузку и должно отображать процесс, которую он реализует.
   При написании наименования следует придерживать UpperCamelCase

5. Расстановка блоков
   	if (условие)
    {
     	операция
    } 
	  else
    {
	    операция
    } 

6. Оформление арифмитических выражений:
	Каждое выражение должно записываться в новой строке.

7. Оформление комментариев:
	Каждую функцию необходимо предворять комментариями с описанием на русском языке ее целей и задач.

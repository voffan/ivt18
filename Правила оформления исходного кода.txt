Правила оформления исходного кода:

1. Именование Классов:
   При именовании следует придерживаться UpperCamelCase { ExampleClass }

2. Именование переменных/данных(поле) класса
   При именовании следует придерживаться lowerCamelCase { exampleVariable }

3. Именование свойств класса:
   При именовании следует придерживаться UpperCamelCase

4. Именование функций/методов класса:
   При именовании следует придерживаться UpperCamelCase

5. Расстановка блоков
   При оформлении логических блоков следует придерживаться стиля Олмана:
    if (<..>)
    {
        <..>
    }

6. Оформление арифмитических выражений:
   Каждый оператор должен отделяться пробелом от своих аргументов
    { i = 0; i < n; i++ }

7. Оформление комментариев:
(комментарии перед функциями)
    /*
        @return void
        kto? - kogda?
        chto eto?
    */
